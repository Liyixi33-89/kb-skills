# Deploy-Check — 部署前检查

## 目标

在代码部署到目标环境之前，自动执行全面的检查清单，确保部署安全、完整、可回滚。输出"部署就绪报告"，明确标注通过/未通过/警告项。

**核心隐喻**：你是一个"飞行前检查员"——飞机起飞前必须完成所有检查项，任何一项关键检查未通过都不能起飞。宁可延迟发布，也不能带着问题上线。

---

## 设计原则

1. **清单驱动**：所有检查项预定义，逐项执行，不遗漏
2. **分级告警**：区分 🔴 阻塞部署（必须修复）、🟡 警告（建议修复但不阻塞）、🟢 信息（仅供参考）
3. **环境感知**：根据目标环境（dev/staging/production）调整检查严格度
4. **可回滚验证**：确认回滚方案可行
5. **自动化优先**：能自动检查的不手动，能自动修复的给出修复命令

---

## 检查维度

| 维度 | 检查项数 | 说明 |
|------|---------|------|
| **环境变量** | 5+ | .env 文件完整性、敏感信息不泄露 |
| **依赖版本** | 3+ | package.json 一致性、lock 文件同步 |
| **构建产物** | 4+ | 构建成功、产物大小合理、无源码泄露 |
| **数据库** | 3+ | 迁移脚本状态、Schema 兼容性 |
| **API 兼容性** | 3+ | 破坏性变更检查、版本号更新 |
| **安全** | 4+ | 依赖漏洞、敏感信息扫描 |
| **测试** | 2+ | 测试通过率、覆盖率 |
| **文档** | 2+ | CHANGELOG 更新、API 文档同步 |

## 编排流程

### 第 1 步：环境变量检查

#### 1.1 .env 文件完整性

读取项目中的 `.env.example`（或 `.env.template`），对比目标环境的 `.env` 文件：

| 检查项 | 级别 | 说明 |
|--------|------|------|
| 所有必填变量都有值 | 🔴 | 缺少必填变量会导致启动失败 |
| 无多余的未使用变量 | 🟢 | 清理无用配置 |
| 数据库连接字符串格式正确 | 🔴 | 格式错误会导致连接失败 |
| JWT_SECRET 长度 ≥ 32 字符 | 🟡 | 安全建议 |
| 敏感变量不在代码中硬编码 | 🔴 | 安全要求 |

#### 1.2 环境特定检查

| 环境 | 额外检查 |
|------|---------|
| production | NODE_ENV=production、DEBUG=false、日志级别 ≤ warn |
| staging | 数据库指向 staging 库（非 production） |
| dev | 无额外要求 |

### 第 2 步：依赖版本检查

| 检查项 | 级别 | 说明 |
|--------|------|------|
| `package-lock.json` 与 `package.json` 同步 | 🔴 | 不同步可能导致依赖版本不一致 |
| 无 `npm audit` 高危漏洞 | 🟡 (staging) / 🔴 (production) | 安全要求 |
| Node.js 版本匹配 `engines` 字段 | 🔴 | 版本不匹配可能导致运行时错误 |

**检查命令**：
```bash
npm audit --production
npm ls --depth=0  # 检查依赖树
node -v           # 检查 Node 版本
```

### 第 3 步：构建产物检查

| 检查项 | 级别 | 说明 |
|--------|------|------|
| `npm run build` 成功完成 | 🔴 | 构建失败不能部署 |
| 构建产物大小合理（前端 bundle < 5MB） | 🟡 | 过大影响加载速度 |
| 构建产物不包含 `.env` 文件 | 🔴 | 安全要求 |
| 构建产物不包含 `node_modules` | 🟡 | 部署包过大 |
| TypeScript 编译无错误 | 🔴 | 类型错误可能导致运行时问题 |

### 第 4 步：数据库检查

| 检查项 | 级别 | 说明 |
|--------|------|------|
| 所有迁移脚本已执行 | 🔴 | 未执行的迁移会导致 Schema 不匹配 |
| 迁移脚本有回滚方法 | 🟡 | 确保可回滚 |
| 新增索引不影响现有查询性能 | 🟡 | 大表加索引可能锁表 |
| 数据库连接池配置合理 | 🟢 | 性能建议 |

**检查方式**：
- 读取 `server/src/migrations/` 目录，对比 `migration-log.md`
- 检查是否有未执行的迁移脚本

### 第 5 步：API 兼容性检查

| 检查项 | 级别 | 说明 |
|--------|------|------|
| 无破坏性 API 变更（删除端点、修改参数） | 🔴 (production) | 会影响现有客户端 |
| 新增 API 有认证中间件 | 🔴 | 安全要求 |
| API 版本号已更新（如有破坏性变更） | 🟡 | 版本管理 |
| OpenAPI 文档已同步更新 | 🟡 | 文档一致性 |

**检查方式**：
- 对比 KB 中的 `01_index_api.md` 与当前路由文件
- 检查是否有删除或修改签名的端点

### 第 6 步：安全检查

| 检查项 | 级别 | 说明 |
|--------|------|------|
| 无硬编码的密码/Token/API Key | 🔴 | 安全要求 |
| CORS 配置正确（production 不允许 *） | 🔴 (production) | 安全要求 |
| Rate Limiting 已配置 | 🟡 (production) | 防止滥用 |
| HTTPS 强制（production） | 🔴 (production) | 安全要求 |

**检查方式**：
- 扫描代码中的敏感字符串模式（`password=`、`secret=`、`api_key=`）
- 检查 CORS 中间件配置
- 检查 Rate Limiting 中间件

### 第 7 步：测试检查

| 检查项 | 级别 | 说明 |
|--------|------|------|
| 所有测试通过 | 🔴 | 测试失败不能部署 |
| 测试覆盖率 ≥ 60% | 🟡 (staging) / 🔴 (production) | 质量要求 |

**检查命令**：
```bash
npm test -- --coverage
```

### 第 8 步：文档检查

| 检查项 | 级别 | 说明 |
|--------|------|------|
| CHANGELOG.md 已更新 | 🟡 | 版本追踪 |
| API 文档与代码同步 | 🟡 | 文档一致性 |
| KB 已更新 | 🟢 | 知识库同步 |

## 约束

### 检查约束

1. **Production 环境最严格**：所有 🟡 警告在 production 环境升级为 🔴 阻塞
2. **不修改代码**：检查只输出报告，不直接修改代码或配置
3. **回滚方案必须可行**：每个部署报告都必须包含回滚方案
4. **检查可重复执行**：多次执行结果一致（幂等性）

### 边界条件处理

| 场景 | 处理方式 |
|------|---------|
| .env.example 不存在 | 标注 `⚠️ 无环境变量模板`，跳过环境变量检查 |
| 无测试文件 | 标注 `🔴 无测试覆盖`，建议先执行 `gen-test-code` |
| 无迁移脚本 | 标注 `🟢 无数据库变更` |
| 无 CHANGELOG | 标注 `🟡 CHANGELOG 未更新`，建议执行 `changelog-gen` |
| Git 仓库不存在 | 跳过 git diff 相关检查 |
